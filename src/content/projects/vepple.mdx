## Overview

Vepple is a virtual experience platform used by 30+ UK universities including Imperial College London, Nottingham Trent, and Queen's University Belfast. I led front-end and UX development at Revolution Viewing, collaborating directly with the CTO and Product Manager on product direction and supporting customer success through onboarding, demos, and feedback sessions.

I'm currently spearheading a ground-up front-end rebuild with the CTO and Product Manager, replacing the Vue.js/Quasar stack with a production-grade React, Next.js 16, TypeScript, and Payload CMS codebase with full CI/CD and automated testing.

## Key contributions

### Product surfaces

**Mobile navigation.** The original header-based navigation wasn't cutting it on mobile. I designed and built a completely new system from scratch: a fixed bottom tab bar with five contextual actions: Menu, Map, Personalise, Checklist, and Chat. Each tab opens custom swipeable bottom sheet panels built with [VueUse](https://vueuse.org), surfacing expanded navigation, Open Day schedule tracking, filter controls, and direct access to Explore by Map. The result: **85% increase** in mobile engagement and **41% longer** session duration.

**Events management system.** Universities needed a way to host hybrid virtual and on-campus open days. I architected a comprehensive events system with full calendar views (day/week/month), [Firebase](https://firebase.google.com) Firestore-backed personal agendas, real-time countdown timers, iCal export, and URL-persisted filters. Built TikTok-inspired session pages to cycle through open day talks with swipeable navigation. Integrated Firebase real-time chat alongside Q&A platforms like [Slido](https://www.slido.com) and [Chatify](https://chatify.com). Drove a **108% increase** in views per session, with users staying **3+ minutes longer** on average.

**Explore by Map.** Built interactive maps using [Mapbox GL JS](https://www.mapbox.com/mapbox-gljs) with custom 3D/2D controls, turn-by-turn directions (walking, cycling, driving), geolocation, location clustering with grouped pins, and fuzzy search. Map users now average **21 pages per session**, with **+68% page depth** and **+64% engagement time** compared to non-map users.

**Guided Tours.** Enhanced the Guided Tour suite built on [Pannellum](https://pannellum.org) for 360° panoramas, adding interactive hotspots, intro/end screens featuring ambassador videos with real-time captioning, mini-map navigation, and content spots embedding reels, galleries, and videos. Added dynamic FOV logic that adjusts zoom bounds based on panorama aspect ratio. Over **500K students** have used Guided Tours, viewing **65% more pages**, staying **50% longer** per session, and rating the experience **4.3 stars**.

**Home screen designs.** Architected a modular home page system with four interchangeable designs, configurable via each client's dedicated WordPress instance with GraphQL. Variants include a typewriter animation with layered opacity, gesture-based filter panels with glassmorphism styling, and an embedded Guided Tour preview reusing the Pannellum viewer and ambassador video captioning system.

### Platform foundations

**Component library.** Built reusable content blocks for universities to customise their experiences, documented in [Histoire](https://histoire.dev) (Storybook for Vue). Blocks include carousels, video embeds, testimonials, and call-to-action sections that universities can configure via their CMS.

**Headless CMS architecture.** Developed custom [ACF](https://www.advancedcustomfields.com) blocks and GraphQL queries for the headless WordPress CMS using [WPGraphQL](https://www.wpgraphql.com). Each university has a dedicated WordPress instance where content editors configure their experience, with the Vue.js frontend fetching content via GraphQL through [Stellate](https://stellate.co) CDN for edge caching. I write queries and mutations across both the CMS and frontend, ensuring performant content delivery for editors and end users alike.

### Analytics and accessibility

**Analytics and A/B testing.** Improved the GTM analytics infrastructure across the frontend and shared component library. Built a centralised data layer module to capture user interactions, enabling the marketing team to measure engagement with video content, interactive tours, and navigation flows. On top of it, built a production A/B testing framework on the [GrowthBook](https://www.growthbook.io) SDK, running **15 experiments across 20K+ users** with a **+19.2% click rate uplift** across home page variants, guided tour UI, and navigation flows, all with GDPR-compliant tracking.

**Accessibility mode.** Architected a comprehensive WCAG-compliant accessibility layer with centralised [Pinia](https://pinia.vuejs.org) state management, supporting OS preference detection and shareable accessible URLs via query parameters. When enabled, accessibility mode activates global audio-described video track switching across embedded video platforms, auto-expanding transcripts, and 360° panorama motion suppression. Integrated [@vue-a11y/announcer](https://vue-a11y.com/announcer) for screen reader live region announcements, with a keyboard-navigable toggle, visible focus states, and analytics tracking.

## Reach

Beyond the headline numbers, students average 12.13 pages and 6m52s per visit, with international visitors making up **40% of traffic** from over 200 countries. **70% of students** said Vepple makes them more likely to attend Open Day; **65%** said it makes them more likely to apply. Guided tours led by students see a **155% engagement uplift** vs platform average. At the University of Salford, "Day in the Life" tours delivered **44% longer engagement** and **111% more page views** per session. For Birmingham City University, Vepple represented over **10% of total CRM engagements**, rivalling high-touch recruitment events.
